## **FormProvider**

`<FormProvider />` is responsible for passing down the theme to the all the form components. It expects the theme to be of certain structure.


---

## **Input**

Input component grabs the theme given to `<FormProvider />` and applies the style defined there based on the given `invalid` prop value or its state. You can use any props you would use on a regular input (i.e. the props are spread over the `<input />` element)

<Props of={Input} />


---


## **InlineInput**

InlineInput component grabs the theme given to `<FormProvider />` and applies the style defined there based on the given `invalid` prop value or its state. You can use any props you would use on a regular input (i.e. the props are spread over the `<input />` element). Since forms with inline inputs do not usually have submit buttons (i.e. inline editing), it offers a few other custom props to deal with the loading state: `submitting` and `renderSpinner`.

<Props of={InlineInput} className="max-width-m" />


---


## **Textarea**

Textarea component grabs the theme given to `<FormProvider />` and applies the style defined there based on the given `invalid` prop value or its state. You can use any props you would use on a regular textarea (i.e. the props are spread over the `<textarea />` element)

<Props of={Textarea} />


---


## **Select**

Select component grabs the theme given to `<FormProvider />` and applies the style defined there based on the given `invalid` prop value or its state. You can use any props you would use on a regular select (i.e. the props are spread over the `<select />` element)

<Props of={Select} />


---


## **Radio**

Radio component grabs the theme given to `<FormProvider />` and applies the style defined there based on the given `invalid` prop value or its state. You can use any props you would use on a regular radio input (i.e. the props are spread over the `<input type="radio" />` element)

<Props of={Radio} />


---


## **Checkbox**

Checkbox component grabs the theme given to `<FormProvider />` and applies the style defined there based on the given `invalid` prop value or its state. You can use any props you would use on a regular checkbox input (i.e. the props are spread over the `<input type="checkbox" />` element)

<Props of={Checkbox} />


---


## **Toggle**

Checkbox component grabs the theme given to `<FormProvider />` and applies the style defined there under the toggle key. Since there is no native toggle component, it only accepts the following props:

<Props of={Toggle} />
